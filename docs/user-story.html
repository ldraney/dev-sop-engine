User Stories

Primary Personas

1. Solo Developer

Wants to standardize their Claude workflow across projects without manually copying config files.

2. Team Lead

Wants to enforce consistent Claude configuration across team members and projects.

3. Open Source Maintainer

Wants to include Claude configuration in their repo so contributors have a consistent experience.

User Stories

New Project Setup

As a developer starting a new project

I want to run npx dev-sop-engine init

So that I get a starter sop.json and sop/ scaffold to customize

$ npx dev-sop-engine init
Created sop.json
Created sop/CLAUDE.md
Created sop/hooks/.gitkeep
Created sop/skills/.gitkeep

Run 'npx dev-sop-engine .' to generate .claude/

Generate Configuration

As a developer with an existing sop.json

I want to run npx dev-sop-engine .

So that I get an updated .claude/ and .mcp.json

$ npx dev-sop-engine .
Reading sop.json...
Generated .claude/settings.json
Generated .claude/CLAUDE.md
Copied .claude/hooks/lint.sh
Generated .claude/skills/workflow/SKILL.md
Generated .mcp.json

Done. 5 files written.

Share Configuration with Team

As a team lead

I want to commit sop.json and sop/ to our repo

So that team members can run npx dev-sop-engine . and get identical Claude configuration

Inject into Existing Project

As a developer

I want to generate .claude/ in a project that doesn't use dev-sop-engine

So that I can use my standard Claude configuration anywhere

$ cd ~/my-other-project
$ npx dev-sop-engine ~/my-sop-config
Generated .claude/ from ~/my-sop-config/sop.json

Portable Output for Contributors

As an open source maintainer

I want to commit the generated .claude/ directory

So that contributors don't need dev-sop-engine installed

Validation

As a developer

I want to validate my sop.json before generating

So that I catch configuration errors early

$ npx dev-sop-engine validate
sop.json is valid.

Non-Goals (v1)

  • GUI or web interface
  • Plugin system
  • Watch mode / auto-regenerate
  • Diff / preview mode
  • MCP server mode (generate-only CLI for v1)
